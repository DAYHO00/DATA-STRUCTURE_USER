## HW1. 스택 기초 및 응용

### 과제 개요
자료구조 **스택(Stack)**의 기본 개념을 이해하고, LIFO(Last-In-First-Out) 구조를 활용한 문제 해결 능력을 기른다.

### 핵심 내용
- 스택 ADT 구현
- push / pop 연산의 동작 원리
- 스택을 이용한 상태 추적 및 시뮬레이션
- 시간 복잡도 분석

## HW2. 괄호 값 계산

### 문제 설명
`()`, `[]` 로 이루어진 문자열이 주어질 때, 다음 규칙에 따라 **괄호 문자열의 값을 계산**한다.

- `()` → 3  
- `[]` → 7  
- `(X)` → 3 × X  
- `[X]` → 7 × X  
- `XY` → X + Y  
- 올바르지 않은 괄호 구조 → 0 반환

### 구현 조건
- **연결 리스트 기반 스택 사용**
- **재귀 사용 금지**
- 정수 결과만 출력

### 해결 방법
- 스택을 이용해 괄호의 짝을 검사
- 괄호가 닫힐 때마다 값을 계산하여 누적
- 잘못된 구조가 나오면 즉시 종료

### 복잡도
- 시간 복잡도: `O(N)`
- 공간 복잡도: `O(N)`

## HW3. 혼잡도를 고려한 최단 경로

### 문제 설명
회사와 직원 집을 노드로 갖는 그래프에서 **도로의 혼잡 시간대에 따라 가중치가 달라지는 상황**을 고려하여 최단 도착 시간을 계산한다.

### 핵심 아이디어
- 기본 구조는 **다익스트라(Dijkstra) 알고리즘**
- 간선 가중치는 시간대에 따라 동적으로 결정
- 우선순위 큐(최소 힙)를 이용해 효율적으로 탐색

### 특징
- 이동 중 정지 불가
- 혼잡 시간 구간을 고려한 비용 계산

### 복잡도
- 시간 복잡도: `O((V + E) log V)`
- 공간 복잡도: `O(V)`

## HW4. 스택·그래프 응용 문제

### 포함 문제

#### 1) Train Rearrangement
- T자형 철도 구조에서 기차를 1~N 순서로 재배열 가능한지 판단
- (b) 구역은 **스택(LIFO)** 역할

**해결 방법**
- 스택 시뮬레이션
- 현재 필요한 기차 번호를 기준으로 push / pop 수행

#### 2) Agony of Engineer
- 두 건물 간 단자 연결 관계를 복원
- **이동 횟수를 최소화**하는 것이 핵심 목표
- 주어진 API를 활용하여 연결 상태 추론

## HW5. 트리 알고리즘

### 문제 구성

#### 1) One Fire Station
- 트리 구조의 도시에서 소방서 1개를 배치
- 모든 건물까지의 **최대 거리 최소화**

#### 2) Two Fire Stations
- 소방서 2개 배치
- 트리의 지름을 분할하여 균형 있게 배치

### 핵심 알고리즘
- BFS를 이용한 **트리 지름(diameter) 계산**
- 경로의 중앙을 기준으로 소방서 위치 선정

### 복잡도
- 시간 복잡도: `O(N)`
- 공간 복잡도: `O(N)`

## HW6. 난수 생성 알고리즘

### 알고리즘

#### 1) XORShift
- XOR + 비트 시프트 기반 난수 생성
- 매우 빠르지만 암호학적 안전성은 낮음

#### 2) Blum Blum Shub (BBS)
- 암호학적으로 안전한 난수 생성기
- 모듈러 제곱 기반, 연산 비용 큼

#### 3) WELL (Well-Equidistributed Long-period Linear)
- Mersenne Twister 개선형
- 균등 분포와 긴 주기 제공
- 메모리 사용량이 비교적 큼

### 비교 관점
- 속도
- 보안성
- 주기 길이
- 사용 목적별 적합성
